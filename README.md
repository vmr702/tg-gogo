# 🌆 CityEvents Bot — Локальная Афиша Мероприятий

Телеграм-бот для поиска и добавления городских событий. Проект построен на базе **Slim 4 Framework**, **Eloquent ORM** и **PostgreSQL 16** в изолированном **Docker**-окружении.

## 🎯 Идея и Функционал
Бот помогает пользователям находить актуальные события в своем городе и делиться собственными мероприятиями с ограничением на спам.

*   **Локализация по городу:** При старте (`/start`) бот предлагает выбрать город. Выбранный город сохраняется в профиль пользователя (используется поле `city` вместо `first_name`).
*   **Пагинация (Показать еще):** Мероприятия выводятся порциями по 10 штук. Внизу списка доступны инлайн-кнопки управления. Нажатие «Показать еще» динамически подгружает следующие 10 записей.
*   **Просмотр деталей:** Каждому событию в списке присваивается порядковый номер (1-10). При отправке цифры бот открывает полную карточку мероприятия.
*   **Защита от спама (Анти-флуд):** Любой пользователь может добавить свое событие в общую ленту города, но **не чаще 1 раза в час**. Ограничение валидируется на уровне базы данных.

## 🛠 Технологический стек
*   **PHP 8.3-fpm** (на базе дистрибутива Alpine)
*   **Slim 4** — ядро для обработки входящих Webhook-запросов от Telegram
*   **PostgreSQL 16** — надежное хранилище данных
*   **Eloquent ORM (v10)** — управление моделями данных и связями без SQL-запросов
*   **Longman Telegram Bot API** — библиотека для отправки сообщений и построения клавиатур
*   **vlucas/phpdotenv** — безопасное управление конфигурацией через `.env` файлы

## 📊 Архитектура Базы Данных

### Таблица `users`
*   `id` (Primary Key)
*   `telegram_id` (bigint, unique) — уникальный ID пользователя в Telegram
*   `username` (string, nullable) — юзернейм в Telegram
*   `city` (string, nullable) — выбранный город пользователя (используется вместо `first_name`)
*   `created_at / updated_at`

### Таблица `events`
*   `id` (Primary Key)
*   `user_id` (Foreign Key) — ID автора, создавшего событие
*   `city` (string) — город, к которому относится мероприятие
*   `title` (string) — название события
*   `description` (text) — подробное описание (дата, время, место, контакты)
*   `created_at / updated_at`

## 🚀 Быстрый старт

### 1. Настройка переменных окружения
Склонируйте репозиторий и создайте локальный файл конфигурации:
```bash
cp .env.example .env
```
Откройте созданный файл `.env` и добавьте ваш токен бота, полученный от `@BotFather`:
```ini
DB_HOST=db
DB_NAME=bot_db
DB_USER=bot_user
DB_PASSWORD=bot_pass
TG_TOKEN=ВАШ_ТОКЕН_БОТА
```

### 2. Развертывание Docker-контейнеров
Запустите базу данных и PHP-сервис в фоновом режиме:
```bash
docker compose up -d --build
```

### 3. Установка зависимостей и миграции
Установите пакеты Composer и создайте структуру таблиц в PostgreSQL:
```bash
docker compose exec php composer install
docker compose exec php php migrate.php
```

### 4. Включение локального веб-сервера
Запустите встроенный сервер внутри контейнера для обработки вебхуков:
```bash
docker compose exec -d php php -S 0.0.0.0:8080 -t public
```

## 💻 Разработка и тестирование Webhook
Так как сервера Telegram не могут отправлять запросы на `localhost`, используйте утилиту **ngrok** для проброса локального порта наружу:

1. Запустите туннель: `ngrok http 8080`
2. Скопируйте выданный вам адрес `https://...`
3. Зарегистрируйте Webhook в Telegram, открыв в браузере ссылку:
   `https://telegram.org<ВАШ_TG_TOKEN>/setWebhook?url=https://<ВАШ_АДРЕС_NGROK>/webhook/telegram`
