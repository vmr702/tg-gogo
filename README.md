# 🌆 CityEvents Bot — Локальная Афиша Мероприятий

Телеграм-бот для поиска и добавления городских событий. Проект построен на базе **Slim 4 Framework**, **Eloquent ORM** и **PostgreSQL 16** в изолированном **Docker**-окружении.

---

## 🎯 Реализованный функционал (MVP)

*   **Локализация по городу:** При старте (`/start`) бот предлагает инлайн-кнопки для выбора города. Выбранный город записывается в профиль пользователя в поле `city`.
*   **Динамическая пагинация:** Мероприятия выводятся порциями по 10 штук с нумерацией. Под списком отображаются инлайн-кнопки `🔄 Показать еще`, `🔍 Фильтры` и `➕ Добавить новое`.
*   **Просмотр деталей по цифре:** При отправке текстовой цифры (например, `3`) бот проверяет текущий город пользователя, находит соответствующее мероприятие в БД и выводит его полное описание.
*   **Связи данных (Relations):** На уровне СУБД настроена правильная реляционная связь: мероприятия в таблице `events` жестко привязаны к внутреннему `id` авторов из таблицы `users` (`foreignId` с каскадным удалением).
*   **Безопасность среды:** Конфиденциальные токены и доступы к БД вынесены в изолированный файл `.env` и считываются через пакет `vlucas/phpdotenv`.

## 🛠 Технологический стек

*   **Язык:** PHP 8.3-fpm (Alpine)
*   **Ядро:** Slim 4 Framework (обработка входящих POST-вебхуков)
*   **База данных:** PostgreSQL 16
*   **ORM-слой:** Illuminate Database (Eloquent v10)
*   **API Telegram:** Longman Telegram Bot API (v0.81)
*   **Утилиты окружения:** Docker, Docker Compose, vlucas/phpdotenv
*   **Локальный туннель:** ngrok (для проброса https-трафика от серверов Telegram)

## 📊 Структура Базы Данных

### Таблица `users`
*   `id` (Bigserial, Primary Key) — внутренний инкрементный ID
*   `telegram_id` (bigint, unique) — уникальный ID аккаунта в Telegram
*   `username` (string, nullable) — юзернейм пользователя
*   `city` (string, nullable) — сохраненный город

### Таблица `events`
*   `id` (Bigserial, Primary Key)
*   `user_id` (Foreign Key -> `users.id`) — ID автора, создавшего событие (с индексацией `onDelete('cascade')`)
*   `city` (string) — город проведения (проиндексирован для быстрого поиска)
*   `title` (string) — название события
*   `description` (text) — дата, время, место и контакты
*   `created_at / updated_at` (timestamps)

## 🚀 Инструкция по развертыванию

### 1. Настройка конфигурации
Склонируйте репозиторий и создайте локальный рабочий конфиг:
```bash
cp .env.example .env
```
В файле `.env` укажите реальный `TG_TOKEN` вашего бота от `@BotFather`.

### 2. Сборка Docker и установка пакетов
```bash
# Запуск контейнеров в фоне
docker compose up -d --build

# Установка зависимостей Composer внутри контейнера
docker compose exec php composer install
```

### 3. Накат структуры БД и сидов
```bash
# Создание таблиц migrations, users и events
docker compose exec php php migrate.php

# Наполнение базы тестовыми мероприятиями для Москвы
docker compose exec php php seed.php
```

### 4. Включение вебхука
Запустите встроенный сервер PHP для прослушивания порта в контейнере:
```bash
docker compose exec -d php php -S 0.0.0.0:8080 -t public
```
Пробросьте локальный порт наружу через ngrok (`ngrok http 8080`), скопируйте сгенерированный `https` адрес и установите Webhook в Telegram:
```text
https://telegram.org<ВАШ_TG_TOKEN>/setWebhook?url=https://<ВАШ_АДРЕС_NGROK>/webhook/telegram
```
